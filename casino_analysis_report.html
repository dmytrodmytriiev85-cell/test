1. Фреймворк и архитектура

Обнаруженная технология: Next.js

Веб-сайт Lucky Hands Casino построен на фреймворке Next.js, который является production-ready фреймворком для React приложений с поддержкой Server-Side Rendering и Static Site Generation.

Признаки Next.js в исходном коде:

  • Наличие директории /_next/static/ для статических файлов
  • Next.js CSS файлы с путями app директории
  • Next.js JavaScript бандлы (main-app.js, app-pages-internals.js)
  • Webpack бандлер (встроенный в Next.js)

HTML структура и Next.js элементы:

Next.js CSS файлы в HTML
<link rel="stylesheet" href="/_next/static/css/app/%5Blocale%5D/layout.css?v=1777627214283"
    data-precedence="next_static/css/app/[locale]/layout.css"/>

<link rel="preload" href="/_next/static/css/app/%5Blocale%5D/page.css?v=1777627214283"
    as="style"/>
Next.js JavaScript бандлы
<script src="/_next/static/chunks/main-app.js?v=1777627214283"></script>
<script src="/_next/static/chunks/app-pages-internals.js"></script>
<script src="/_next/static/chunks/app/%5Blocale%5D/loading.js"></script>
<script src="/_next/static/chunks/app/%5Blocale%5D/layout.js"></script>
<script src="/_next/static/chunks/polyfills.js"></script>
<script src="/_next/static/chunks/webpack.js"></script>

Архитектурные характеристики:

Характеристика Значение

2. Компонентные библиотеки

Обнаруженное решение: Собственная система компонентов

Анализ исходного кода показал, что веб-сайт не использует готовые компонентные библиотеки (Material-UI, Chakra UI, shadcn/ui и т.д.). Вместо этого используется собственная система компонентов.

Проверенные и не обнаруженные библиотеки:

Библиотека Статус Причина
Material-UI (MUI) ❌ Не используется Нет импортов @mui/*
Chakra UI ❌ Не используется Нет импортов @chakra-ui/*
shadcn/ui ❌ Не используется Нет импортов shadcn/*
Ant Design ❌ Не используется Нет импортов antd
Radix UI ❌ Не используется Нет импортов @radix-ui/*
Bootstrap ❌ Не используется Нет импортов bootstrap

Преимущества собственной системы:

  • Полный контроль над UI/UX и брендингом
  • Оптимизация размера бандла
  • Специфические компоненты для казино
  • Кастомная дизайн система согласно видению бренда

3. Дизайн система и CSS токены

Обнаруженное решение: CSS переменные (CSS Custom Properties)

Веб-сайт использует CSS переменные для реализации дизайн системы с токенами. Это позволяет централизованно управлять цветами, размерами и другими значениями стилей.

Найденные CSS переменные (токены):

Токен Тип Примечание
--lh-secondary Цвет Вторичный цвет бренда Lucky Hands
--headerHeight Размер Высота шапки сайта
--containerWidth Размер Максимальная ширина контейнера
--header Прочее Переменная связанная с header

Использование CSS переменных в коде:

Примеры использования var() в CSS
/* Инлайн стили с использованием переменных */
style={{ color: 'var(--lh-secondary)' }}
style={{ marginTop: 'var(--headerHeight)' }}
style={{ maxWidth: 'var(--containerWidth)' }}

Архитектура CSS:

Аспект Реализация
CSS переменные ✓ Используются
Tailwind CSS ❌ Не используется
SCSS переменные ❌ Не обнаружены
CSS Modules ✓ Вероятно используются

Загруженные CSS файлы:

CSS файлы приложения
1. /_next/static/css/app/[locale]/layout.css
